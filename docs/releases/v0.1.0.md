# v0.1.0 — Safe Windows read-only baseline

This release establishes the first public baseline for **ubertooth-windows-host**.

## Highlights
- Detect Ubertooth One on Windows
- Validate WinUSB readiness and interface access
- Read official firmware/build/board/serial/part info
- Read official getter-only runtime state
- Expose a single `status` summary command for end users

## Verified
- `npm run check`
- `npm run status` on connected hardware

## Current safety boundary
This release is intentionally **read-only only**:
- no control-out writes
- no DFU
- no flashing
- no mode changes

## Known limitations
- No sniffing/capture path yet
- No flashing path
- No installer/release packaging beyond repo workflows
- Branch protection still needs manual GitHub configuration
